# Code of Conduct

## Our Standard

This project is for designing a practical, auditable protocol for multi-agent
collaboration. Contributions should be technical, evidence-backed, and
respectful.

Examples of expected behavior:

- focus on the protocol, evidence, and implementation tradeoffs;
- assume good faith while asking for concrete proof;
- separate facts from opinions;
- document risks and limitations clearly;
- avoid personal attacks, harassment, or discriminatory language.

Examples of unacceptable behavior:

- harassment, insults, threats, or sustained personal attacks;
- publishing private information without permission;
- knowingly submitting malicious instructions, secrets, or unsafe automation;
- derailing technical discussion with bad-faith arguments.

## Enforcement

Maintainers may remove comments, issues, pull requests, or contributions that
violate this code of conduct. Repeated or severe violations may result in a ban
from project spaces.

## Reporting

Report conduct concerns through the repository's security or maintainer contact
channels. Do not disclose private information publicly.
